# 村田鉄筋㈱ 取付数量分析アプリ

## プロジェクト概要
- **アプリ名**: 村田鉄筋㈱ 取付数量分析アプリ
- **目的**: 元請・現場ごとに鉄筋の取付数量・人工数を記録し、現場別の合計数量・合計人工数・1人工あたりの取付数量を分析する業務アプリ
- **設計方針**: **「現場を先に登録」→「その現場に取付実績を追加」** という二段構えのフロー
- **技術スタック**: Hono + TypeScript + Cloudflare Pages + Cloudflare D1 (SQLite) + TailwindCSS + Chart.js

## データ登録の流れ
```
[1] 現場一覧画面で「現場を追加」
        ↓
    元請名・現場名・備考を入力して登録
        ↓
[2] 登録した現場をクリック
        ↓
    現場詳細画面が開き、その現場専用の取付実績を追加・編集・削除
        ↓
[3] ダッシュボード / 集計・分析画面で全体や絞り込みでの集計を確認
```

## 画面構成

### 1. 現場一覧（タブ: 現場一覧）
- 現場カード形式で表示（モバイルでも見やすい）
- 各カードに **取付日数・合計数量・合計人工数・1人工あたり (kg/人工)** をミニサマリ表示
  - 合計人工数が0の場合は「**未計算**」と表示（ゼロ除算回避）
- **新規登録**：右上「現場を追加」ボタンからモーダルで登録
- **編集**：カードの ✏️ ボタン
- **削除**：カードの 🗑️ ボタン
  - 取付実績がある場合は確認ダイアログで「実績ごと削除」を選べる
- **取付実績入力**：カードの「取付実績を入力／確認」ボタンで現場詳細画面へ
- **検索**：元請名・現場名で部分一致絞り込み

### 2. 現場詳細・取付実績入力（現場カードから開く）
- 上部に元請・現場名・サマリ（取付日数・合計数量・合計人工数・1人工あたり）
- **取付実績入力フォーム**
  - 取付日（必須）
  - 部位（必須、部位マスタから選択）
  - 数量 kg（必須、小数可）
  - 人員 人工（必須、0.5/1.5等 小数可）
  - 人員名（複数チップ入力、過去入力からの datalist 補完あり）
  - 備考
- **取付実績一覧**（その現場のみ表示）
  - 取付日 / 部位 / 数量 / 人員 / 人員名 / 操作
  - **編集**：フォームに転送して更新
  - **削除**：確認ダイアログ付き
- **CSV出力**：その現場の取付実績だけをCSV出力

### 3. ダッシュボード（タブ: ダッシュボード）
- 期間（開始・終了）で絞り込み
- KPIカード：総取付数量 / 総人工数 / 全体の1人工あたり取付数量（未計算表示対応）
- 横棒グラフ：元請別・現場別・部位別・人員別の取付数量

### 4. 集計・分析（タブ: 集計・分析）
- **絞り込み**：期間 / 元請 / 現場名 / 部位 / 人員名
- サマリカード：合計数量・合計人工数・1人工あたり（未計算表示対応）
- **元請・現場ごとのサマリ表**：元請 / 現場名 / 取付日数 / 合計数量 / 合計人工数 / 1人工あたり
- **取付実績一覧**：取付日 / 元請 / 現場名 / 部位 / 数量 / 人員 / 人員名
- **CSV出力**（明細＋サマリ、Excel対応BOM付き）

### 5. 部位設定（タブ: 部位設定）
- 部位の **追加 / 名前変更 / 削除**
- 初期値: 基礎 / 柱 / 梁 / 壁 / スラブ / 土間 / 土木 / その他
- 既存実績の部位文字列はそのまま残る安全設計

## 計算ロジック
| 項目 | 計算式 | 例 |
| --- | --- | --- |
| 合計数量 | 入力された数量 (kg) の合計 | 12,000 kg |
| 合計人工数 | 入力された人員 (人工) の合計 | 6 人工 |
| 1人工あたりの取付数量 | 合計数量 ÷ 合計人工数 | 2,000 kg/人工 |
| 人員別取付数量 | 各取付の数量を作業員数で按分 | - |

**合計人工数が 0 の場合は「未計算」と表示し、エラーになりません。**

## API エンドポイント一覧

### 現場マスタ
| メソッド | パス | 説明 |
| --- | --- | --- |
| GET | `/api/sites` | 現場一覧（集計サマリ付き、`search` クエリで絞り込み） |
| GET | `/api/sites/:id` | 現場1件取得 |
| POST | `/api/sites` | 現場登録（body: `{contractor, site_name, note?}`） |

| GET | `/api/installations/:id` | 取付実績1件取得 |
| POST | `/api/installations` | 取付実績登録（body: `{site_id, work_date, part, quantity, manpower, workers[], note?}`） |
| PUT | `/api/installations/:id` | 取付実績更新 |
| DELETE | `/api/installations/:id` | 取付実績削除 |

### 部位マスタ・補助
| メソッド | パス | 説明 |
| --- | --- | --- |
| GET | `/api/parts` | 部位一覧 |
| POST | `/api/parts` | 部位追加 |
| PUT | `/api/parts/:id` | 部位更新 |
| DELETE | `/api/parts/:id` | 部位削除 |
| GET | `/api/suggestions` | 過去入力した人員名（datalist 補完用） |

### 分析
| メソッド | パス | 説明 |
| --- | --- | --- |
| GET | `/api/analytics/dashboard` | KPI + 元請/現場/部位/人員別の集計 |

## データモデル

### `sites` テーブル（現場マスタ）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| contractor | TEXT NOT NULL | 元請会社名 |
| site_name | TEXT NOT NULL | 現場名 |
| note | TEXT | 備考 |
| created_at / updated_at | DATETIME | タイムスタンプ |
| UNIQUE | (contractor, site_name) | 同じ組み合わせ不可 |

### `installations` テーブル（取付実績）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| **site_id** | INTEGER FK → sites.id | **所属現場** |
| work_date | TEXT | 取付日 (YYYY-MM-DD) |
| contractor | TEXT | 元請（sites と同期） |
| site_name | TEXT | 現場名（sites と同期） |
| part | TEXT | 部位 |
| quantity | REAL | 取付数量 (kg, 小数可) |
| manpower | REAL | 人員 (人工, 小数可) |
| note | TEXT | 備考 |

### `installation_workers` テーブル（人員）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| installation_id | INTEGER FK → installations.id | 取付実績 ID |
| worker_name | TEXT | 人員名 |

### `parts` テーブル（部位マスタ）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| name | TEXT UNIQUE | 部位名 |
| sort_order | INTEGER | 表示順 |

## ユーザーガイド

### 初めて使う場合
1. 右上の「**＋現場を追加**」を押す
2. 元請名（例: 〇〇建設）と現場名（例: 亀岡市〇〇新築工事）を入力して **登録する**
3. 一覧に追加された現場カードの「**取付実績を入力／確認**」を押す
4. 取付日・部位・数量(kg)・人員(人工)・人員名を入力し **追加する**
5. 同じ現場の取付実績はすべてこの画面に一覧で表示されます
6. 「**ダッシュボード**」「**集計・分析**」タブで全体の集計や絞り込み分析ができます

### 絞り込み機能（集計・分析タブ）
- **期間指定**（開始日〜終了日）
- **元請**（登録済み元請から選択）
- **現場名**（登録済み現場から選択）
- **部位**
- **人員名**

### CSV 出力
- **現場詳細画面の「CSV」**：その現場のみの実績
- **集計・分析タブの「CSV出力」**：絞り込んだ全実績＋サマリを 1ファイルに

すべて Excel で文字化けせずに開けます（BOM付き UTF-8）。
