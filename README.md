# 村田鉄筋㈱ 取付数量分析アプリ

## プロジェクト概要
- **アプリ名**: 村田鉄筋㈱ 取付数量分析アプリ
- **目的**: 元請・現場ごとに鉄筋の取付数量・人工数・車両台数を記録し、現場別・部位別の合計数量・合計人工数・1人工あたりの取付数量を分析する業務アプリ
- **設計方針**: **3階層フロー**
  1. **元請・現場名を登録**
  2. その現場の中に **部位・数量 (kg)** を登録
  3. その部位の中に **取付日・人員名・人工数・搬入車両・通勤車両** を登録
- **技術スタック**: Hono + TypeScript + Cloudflare Pages + Cloudflare D1 (SQLite) + TailwindCSS + Chart.js

## データ登録の流れ（3階層）
```
[1] 現場登録画面 (タブ「現場一覧」)
       元請・現場名を登録
           ↓
[2] 部位・数量登録画面 (現場カード「部位・数量を登録／表示」)
       部位（基礎/柱/梁/壁/スラブ/土間/土木/その他 等）と数量(kg) を登録
           ↓
[3] 取付実績登録画面 (部位行「ハンマー」アイコン)
       取付日・人員名（複数可）・人工数・搬入車両・通勤車両 を登録
           ↓
[4] ダッシュボード / 集計・分析画面 で集計を確認
```

### 階層イメージ
```
〇〇建設 / 京都市〇〇マンション新築工事
  └ 基礎 / 5,000kg
      └ 2026/6/18 / 山田、田中 / 2人工 / 搬入1台 / 通勤1台

  └ 柱  / 2,500kg
```

### 7. テキスト取込（タブ「テキスト取込」）★ v4 新機能
予定表のような自由テキストを貼り付けるだけで、**元請・現場名・部位・数量・人工数・人員名・運搬車両** を自動抽出し、3階層構造（現場 → 部位・数量 → 取付実績）に一括登録する画面です。

**画面構成**
- **取付日入力欄**（任意・優先）：空欄の場合はテキスト内の「6/18日の予定です」のような日付を自動採用
- **テキスト貼り付け欄**（10行 textarea、スマホ貼り付け対応、空行で1現場ブロック区切り）
- **読み取りボタン**：`POST /api/import/parse` を呼び、ブロックごとに抽出結果カードを表示
- **クリアボタン**：入力テキストと読み取り結果を破棄
- **読み取り結果一覧**：ブロックごとに編集カード（取付日 / 元請 / 現場名 / 部位 / 数量 / 人工数 / 人員名 / 運搬車両メモ）を表示。**確認画面として動作し、すべて手動で修正可**
- **一括登録ボタン**：確認後に `POST /api/import/commit` を呼び、3階層に upsert
- **キャンセルボタン**：読み取り結果を破棄

**抽出ロジック**
| 項目 | 抽出パターン |
| --- | --- |
| 取付日 | `M/D日(の予定)?` / `M月D日` / `YYYY/M/D`（年なしは今年） |
| 元請 | 各ブロックの先頭セグメント（`、` `,` 区切りの1番目） |
| 現場名 | 元請・日数・数量・人工数の後の最初の非住所・非部位セグメント |
| 部位 | 現場名の次のセグメント（`基礎` / `2F壁` / `枕梁と止水壁` 等そのまま採用） |
| 数量 | `9960K` → 9960 / `約35000K` → 35000（カンマ除去・全角K対応） |
| 人工数 | `16人目` → 16（小数可） |
| 人員名 | `=...` / `応援=...` の後を `、` `,` で分割 |
| 運搬車両メモ | `4トン運搬` / `3トン運搬` / `2トンダンプ` 等を文字列のまま採用 |

**ノイズ自動除外**
- `‼️` `!` `。` `( )` 記号
- 「終わり次第」「第二工場」「終了予定」「8時～朝礼」「13時着」「○○君エブリ」「プロボックス」など
- 住所セグメント（市区町・丁目・番地・付近・号地・擁壁 等）
- 「6/18日の予定です」のような単独ヘッダ行

**読み取り後の処理**
1. **3階層 upsert**（重複検出付き）：
   - `(元請, 現場名)` が一致 → 既存現場に紐付け
   - 同一現場の `(部位)` が一致 → 既存部位に紐付け
   - 新規なら自動作成
2. **数量衝突時の選択**：同部位に異なる数量が既にある場合、ブロックごとに **加算 / 上書き / 既存維持** をラジオで選択（デフォルト=上書き）
3. **取付実績は常に新規登録**：取付日・人員名・人工数・運搬車両メモ（`vehicle_note`） を installations に INSERT。**搬入車両台数 (`delivery_vehicles`) と通勤車両台数 (`commute_vehicles`) は 0 のまま** — 既存の台数入力機能と分離
4. 失敗項目は警告バッジ付きで空欄表示 → 手動で補正

**読み取れない時の運用**
- 警告バッジ「部位を読み取れませんでした」「現場名を読み取れませんでした」等を表示
- そのまま編集欄で手入力 → 「一括登録」で確定
- 部位や現場名が住所形式（番地のみ など）の場合は救済ロジックで現場名候補に採用

**集計への反映**
- テキスト取込で登録されたデータは通常入力と同じ `installations` テーブルに保存されるため、ダッシュボード・集計・分析・部位サマリ・CSV 出力すべてに自動反映
- `kg/人工`、合計数量、合計人工、運搬車両メモなど全項目で従来と同じ動作

### 8. 取付実績の重複登録防止 ★ v5 新機能
取付実績(`installations`)は **同じ現場・同じ部位・同じ取付日** の組み合わせで重複登録できません。

**判定条件 (3つすべて一致した場合に重複と判定)**
- 同じ現場 (`site_id`)
- 同じ部位 (`site_part_id`)
- 同じ取付日 (`work_date`)

**対象**
- 通常入力 (`POST /api/installations`) と編集 (`PUT /api/installations/:id`)
- テキスト取込の一括登録 (`POST /api/import/commit`)

**通常入力でのエラー表示**
- 重複時は HTTP 400 で `{ error, duplicate: true, existing_id }` を返却
- フロントは多行 alert で「元請 / 現場名 / 部位 / 取付日」を提示
- 編集時は自分自身 (`id != ?`) を除外して判定するため、内容を変えずに保存しても重複扱いにならない

**テキスト取込の一括登録での挙動**
- `/api/import/commit` は **1件ずつ結果を返す** (`results: [{index, ok, error?}]`)
- フロントは各ブロックに `commit_status` (`ok` / `ng`) を持たせて表示
  - **成功カード**: 緑バッジ「登録済み」 → 全件成功時はクリア、一部失敗時は画面から除去
  - **失敗カード**: 赤2px枠 + 赤バナーで「登録エラー」とその理由を表示。**画面に残り、修正後に再「DBに登録」可能**
- ユーザが該当カードのフィールドを編集すると失敗状態は自動解除 (`commit_status` を `''` にリセット)
- 「すべて成功」のときだけ読み取り結果画面がクリアされる

**処理イメージ**
```
8件読み取り → 一括登録
   ├ 6件成功 → 緑バッジ「登録済み」 → 画面から除去
   └ 2件失敗 → 赤バナー「すでに同じ取付日の同じ部位が登録されています」
                  → 該当カードのみ画面に残る → ユーザ修正 → 再「DBに登録」
```

## 計算ロジック
| 項目 | 計算式 |
| --- | --- |
| 登録数量 | その部位に登録された数量 (kg) |
| 取付日数 | 取付実績の `work_date` の **DISTINCT** カウント（同日複数件は1日に集約） |
| 合計人工数 | その部位の取付実績の人工数合計 |
| **1人工あたりの取付数量** | **登録数量 ÷ 合計人工数** |
| 搬入車両合計 | 取付実績の搬入車両 (整数) の合計 |
| 通勤車両合計 | 取付実績の通勤車両 (整数) の合計 |

**合計人工数が 0 の場合は「未計算」と表示**（ゼロ除算回避）。

## ユーザーガイド

### 初めて使う場合
1. 「**現場を新規登録**」で元請・現場名を登録
2. 一覧の現場カードで「**部位・数量を登録／表示**」を押す
3. 部位（基礎・柱 など）と数量(kg) を登録
4. 部位行の **🔨** アイコンで取付実績画面を開く
5. 取付日・人員名・人工数・搬入車両・通勤車両を入力 → **追加する**
6. 「ダッシュボード」「集計・分析」で集計を確認

### 数値入力ミス防止
- **人工数 ＋／−0.5 ボタン**：0.5 単位で増減
- **搬入車両／通勤車両 ＋／−1 ボタン**：整数で増減（小数禁止）
- すべて `min=0` で負数防止、`inputmode` でスマホ最適キーボード表示

## 開発・運用コマンド
```bash
# ビルド
npm run build

# サンドボックスで起動（PM2）
pm2 start ecosystem.config.cjs
pm2 logs webapp --nostream

# DB操作（ローカル）
npm run db:migrate:local       # マイグレーション適用 (0001 → 0002 → 0003 → 0004)
npm run db:seed                # サンプルデータ投入
npm run db:reset               # ローカルDBリセット
npm run db:console:local       # ローカルD1コンソール
```
