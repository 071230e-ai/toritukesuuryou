# 村田鉄筋㈱ 取付数量分析アプリ

## プロジェクト概要
- **アプリ名**: 村田鉄筋㈱ 取付数量分析アプリ
- **目的**: 元請・現場ごとに鉄筋の取付数量・人工数・車両台数を記録し、現場別・部位別の合計数量・合計人工数・1人工あたりの取付数量を分析する業務アプリ
- **設計方針**: **3階層フロー**
  1. **元請・現場名を登録**
  2. その現場の中に **部位・数量 (kg)** を登録
  3. その部位の中に **取付日・人員名・人工数・搬入車両・通勤車両** を登録
- **技術スタック**: Hono + TypeScript + Cloudflare Pages + Cloudflare D1 (SQLite) + TailwindCSS + Chart.js

## データ登録の流れ（3階層）
```
[1] 現場登録画面 (タブ「現場一覧」)
       元請・現場名を登録
           ↓
[2] 部位・数量登録画面 (現場カード「部位・数量を登録／表示」)
       部位（基礎/柱/梁/壁/スラブ/土間/土木/その他 等）と数量(kg) を登録
           ↓
[3] 取付実績登録画面 (部位行「ハンマー」アイコン)
       取付日・人員名（複数可）・人工数・搬入車両・通勤車両 を登録
           ↓
[4] ダッシュボード / 集計・分析画面 で集計を確認
```

### 階層イメージ
```
〇〇建設 / 京都市〇〇マンション新築工事
  └ 基礎 / 5,000kg
      └ 2026/6/18 / 山田、田中 / 2人工 / 搬入1台 / 通勤1台
      └ 2026/6/19 / 佐藤、田中 / 2人工 / 搬入0台 / 通勤1台
  └ 柱  / 2,500kg
      └ 2026/6/20 / 山田       / 1人工 / 搬入1台 / 通勤1台
```

## 画面構成

### 1. 現場一覧（タブ「現場一覧」）
- 現場カード形式（スマホで見やすい）
- 各カードに **部位数 / 登録数量合計 / 合計人工 / kg/人工** をミニサマリ表示（人工0なら「未計算」）
- 右上「**現場を新規登録**」モーダル：元請・現場名・備考
- 各カードに **✏️ 編集** / **🗑️ 削除** / **📁 部位・数量を登録／表示**
- **検索**：元請名・現場名で部分一致絞り込み

### 2. 部位・数量登録画面（現場カードから開く）
- 上部に元請・現場名・「現場を編集」
- **追加フォーム**
  - 部位（必須、選択式：部位マスタから）
  - 登録数量 kg（必須、**小数可**、`step="0.01"`）
  - 備考（任意）
- **一覧表**（PCで見やすい表形式）
  - 部位 / 登録数量 (kg) / 取付日数 / 合計人工 / kg/人工 / 搬入車両 / 通勤車両 / 操作
  - **🔨 取付実績へ** / **✏️ 編集** / **🗑️ 削除**（取付実績がある場合は確認ダイアログで強制削除）
- 部位の **追加・編集・削除**：「設定」タブの部位マスタから

### 3. 取付実績登録画面（部位行から開く）
- 上部に **元請 / 現場名 / 部位 / 登録数量 (kg)**
- サマリ：取付日数 / 合計人工 / kg/人工 / 搬入車両合計 / 通勤車両合計（**人工0なら kg/人工 は「未計算」**）
- **入力フォーム**（数値入力ミス防止のため **＋／− ステップボタン** 付き）
  - 取付日（必須）
  - 人員名：チップ複数入力、**過去入力からの datalist 補完**、Enter または「追加」で確定
  - 人工数：**小数可**（`step="0.1"`、＋／−0.5）
  - 搬入車両：**整数のみ**（`step="1"`、＋／−1、min=0）
  - 通勤車両：**整数のみ**（`step="1"`、＋／−1、min=0）
  - 備考
- **取付実績一覧**：取付日 / 人員名 / 人工数 / 搬入車両 / 通勤車両 / 操作（編集・削除）
- **CSV出力**：その部位の取付実績を CSV 出力（Excel 対応 BOM 付き）

### 4. ダッシュボード（タブ「ダッシュボード」）
- 期間（開始・終了）で絞り込み
- KPIカード：**総登録数量 / 総人工数 / kg/人工 / 搬入車両合計 / 通勤車両合計**（未計算表示対応）
- 横棒グラフ：現場別 / 部位別 / 人員別 / 元請別の数量・人工

### 5. 集計・分析（タブ「集計・分析」）
- **絞り込み**：期間 / 元請 / 現場名 / 部位 / 人員名
- **現場一覧** 表：元請 / 現場名 / 合計数量 / 取付日数 / 合計人工 / kg/人工 / 搬入車両 / 通勤車両
- **部位一覧** 表：元請 / 現場名 / 部位 / 登録数量 / 取付日数 / 合計人工 / kg/人工 / 搬入車両 / 通勤車両
- **取付実績一覧** 表：取付日 / 元請 / 現場名 / 部位 / 人員名 / 人工数 / 搬入車両 / 通勤車両
- **CSV出力**（絞り込み後の取付実績一覧、Excel 対応 BOM 付き）

### 6. 部位マスタ（タブ「設定」）
- 部位の **追加 / 名前変更 / 削除**
- 初期値: 基礎 / 柱 / 梁 / 壁 / スラブ / 土間 / 土木 / その他

## 計算ロジック
| 項目 | 計算式 |
| --- | --- |
| 登録数量 | その部位に登録された数量 (kg) |
| 取付日数 | 取付実績の `work_date` の **DISTINCT** カウント（同日複数件は1日に集約） |
| 合計人工数 | その部位の取付実績の人工数合計 |
| **1人工あたりの取付数量** | **登録数量 ÷ 合計人工数** |
| 搬入車両合計 | 取付実績の搬入車両 (整数) の合計 |
| 通勤車両合計 | 取付実績の通勤車両 (整数) の合計 |

**合計人工数が 0 の場合は「未計算」と表示**（ゼロ除算回避）。

## データモデル

### `sites`（現場）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| contractor | TEXT NOT NULL | 元請会社名 |
| site_name | TEXT NOT NULL | 現場名 |
| note | TEXT | 備考 |
| UNIQUE | (contractor, site_name) | 同じ組み合わせ不可 |

### `site_parts`（部位・数量）★ 3階層 第2層
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| **site_id** | INTEGER FK → sites.id | 所属現場 |
| part | TEXT NOT NULL | 部位名 |
| quantity | REAL NOT NULL | 登録数量 (kg, 小数可) |
| note | TEXT | 備考 |
| UNIQUE | (site_id, part) | 同一現場で同一部位の重複不可 |

### `installations`（取付実績）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id | INTEGER PK | 自動採番 |
| **site_part_id** | INTEGER FK → site_parts.id | 所属部位 |
| site_id | INTEGER FK → sites.id | 所属現場（denormalize） |
| work_date | TEXT | 取付日 (YYYY-MM-DD) |
| contractor / site_name / part | TEXT | 集計高速化のためのキャッシュ |
| manpower | REAL | 人工数（小数可） |
| **delivery_vehicles** | INTEGER | 搬入車両台数（整数） |
| **commute_vehicles** | INTEGER | 通勤車両台数（整数） |
| note | TEXT | 備考 |

### `installation_workers`（人員）
| カラム | 型 | 説明 |
| --- | --- | --- |
| installation_id | INTEGER FK | 取付実績 ID |
| worker_name | TEXT | 人員名 |

### `parts`（部位マスタ）
| カラム | 型 | 説明 |
| --- | --- | --- |
| id / name (UNIQUE) / sort_order | - | 部位マスタ |

## ユーザーガイド

### 初めて使う場合
1. 「**現場を新規登録**」で元請・現場名を登録
2. 一覧の現場カードで「**部位・数量を登録／表示**」を押す
3. 部位（基礎・柱 など）と数量(kg) を登録
4. 部位行の **🔨** アイコンで取付実績画面を開く
5. 取付日・人員名・人工数・搬入車両・通勤車両を入力 → **追加する**
6. 「ダッシュボード」「集計・分析」で集計を確認

### 数値入力ミス防止
- **人工数 ＋／−0.5 ボタン**：0.5 単位で増減
- **搬入車両／通勤車両 ＋／−1 ボタン**：整数で増減（小数禁止）
- すべて `min=0` で負数防止、`inputmode` でスマホ最適キーボード表示

### CSV 出力
- 取付実績画面の「CSV」：その部位のみ
- 集計・分析タブの「CSV出力」：絞り込み後の取付実績一覧
